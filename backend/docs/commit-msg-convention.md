# naming convention

| Commit Type  | Meaning                                                                 |
|--------------|-------------------------------------------------------------------------|
| `feat`       | Introduces a new feature                                                |
| `fix`        | Fixes a bug or error                                                    |
| `docs`       | Updates documentation (README, guides, commentsâ€¦)                       |
| `style`      | Changes code formatting or style (e.g., indentation, whitespace)        |
| `refactor`   | Rewrites code without changing external behavior                        |
| `test`       | Adds or modifies tests                                                  |
| `chore`      | Miscellaneous tasks (e.g., package updates, config changes)             |
| `perf`       | Improves performance                                                    |
| `build`      | Changes related to build tools or dependencies                          |
| `ci`         | Updates configuration for Continuous Integration or Deployment          |
| `revert`     | Reverts a previous commit                                               |
